CSS


Three ways to style a webpage using CSS


Inline CSS: when a developer wants to style directly in the HTML file


  1. Apply to a specific HTML tag
  2. Uses a style attribute with a CSS rule to style a particular page element only
  3. Useful for quick/permananent changes
  4. Less flexible than external and internal style sheets...reason each style must be edited individually when making a a design change
  5. Example: using the style attribute to change the color of a single h1 heading

Internal CSS Style Sheet: embedding an entire stylesheet directly into the HTML file using the style element in the head element


  1. Contains rules for the webpage in the head section of the HTML flexible
  2. The rules only apply to that particular page
  3. CSS classes and ID's can be configured to identify and style multiple elements on a page
  4. Similarly to the Inline CSS, a single change to the CSS rule will apply to all tagged page elements

External CSS Style Sheet: most common way to style a webpage


  1. Preferred in most cases because it allows developers to keep all CSS rules in a separate file
  2. Makes design changes easier
  3. when using, we link the file using a link element placed in the HTML's head

Separation of concerns: a widely used term in web development which refers to separating code into different files based on use and different responsibilies


Diagram parts of CSS rules


  1. div: selector, defines an element or attribute to which the rules will apply
    • class attribute: a selector for a new delaration by designation
    • allows to change specific elements but not all
  2. declaration: includes property and value
  3. declaration block: where declarations live, offset by parenthesis

Property Descriptions


  1. *: wild card symbol, it declares all elements on the page will have the same properties
  2. Color: adds a specific color using hex code, rgb code or generic color name
  3. Width and height: defines the space of element
  4. Margin: extra space around the element
  5. Auto: keeps the value to the left and right equal
  6. Border: assigns a border with a defined width and color or hex code
  7. Box-shadow: Adds shadow effects around an elements frame, descrobed by x and y offsets, relative to the element, blur, spread radius and color.
  8. Padding: adds space around the content inside an element
  9. text-align: aligns text right align, center, left align, uniform
  10. card: designates this as a class selector. Any HTML element assigned to the class, card, will gain these property assignments from the CSS

CSS box model: each box consists of margins outside the element, borders, padding, and content


When combining two selectors into a single CSS rule, the formatting applies to both sections.


  1. This saves on repeating the rules twice
  2. Furthermore, two or more elements which share the same style can share the same formatting
  3. Do Not Repeat Yourself (DRY): coding priniciple which encourages developers to reuse/share pieces of code in order to reduce the amount of lines in a code
  4. Helps reduce the overall size and complexity of a codebase

Git


Git Functions and Definitions


  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git config --global user.name "firstname lastname": checks the username set up with account SSH keys
  • git config --global user.email "validemail": checks the email set up with account SSH keys
  • git add file: adds a file as it looks nwo to your nect commit (stage)
  • git reset file: unstage a file while retaining the changes in the working directory
  • git diff: diff of what is changed but not staged
  • git diff --staged: diff of what is staged but not yet committed
  • git commit -m "descriptive message": commit your staged content as a new commit snapshot
  • git branch: list your branches an * will appear next to the currently active branch
  • git branch (branch name): creates new branch at the current commit
  • git checkout: switch to another branch and check it out into your working directory
  • git merge (branch): merge the specified branch's history into the current one
  • git log: show all commits in the current branch's history
  • git init: initialize an existing directory as a Git repository
  • git clone (url): retrieve a repository from a hosted location via url
  • git log: show the commit history for the currently active branch
  • git log branchB..branchA: shwo the commits on branchA that are not on branchB
  • git log --folllow (file): show the commits that changed file, even across renames
  • git diff branchB...branchA: show the diff of what is in branchA that is not in branchB
  • git show: show any object in Git in human-readable format
  • git rm (file): delete the file from project and stage the removal for commit
  • git mv (existing-path) (new-path): change an existing file path and stage the move
  • git log --stat -M: show all commit logs with indication of any paths that moved
  • git remote add (alias) (url): add a git URL as an alias
  • git fetch (alias): fetch down all the branches from that git remote
  • git merge (alias)/(branch): merge a remote branch into your current branch to bring it up to date
  • git push (alias) (branch): transmit local branch commits to the remote repository branch
  • git pull: fetch and merge any commits from the tracking remote branch
  • git rebase (branch): apply any commits of current branch ahead of specified one
  • git reset --hard (commit): clear staging area, rewrite working tree from specified commit
  • git stash: save modified and staged changes
  • git stash list: list stack-order of stashed file changes
  • git stash pop: write working from top of stash stack
  • git stash drop: discard the changes from top of stash stack

JavaScript


Function: a set of instructions that tells the computer how to preform a task


  • Different that a for loop, or conditional statement, because functions do not automatically execute when the JS file (script.js) is run
  • Must call a function by name in order to execute
  • This helps make the code easier to use and replicate/call as many times as needed
  • Also allows to control where in the control flow a task is executed, or if it is at all
  • A .CSS file is linked in the header, meanwhile a .JS is linked in the footer, but they are linked using the same method

2 requirements for a JS function


  1. Define the function/function definition
    • Used through a function declaration
    • Function keyword, the actual word "function" is a prefix to the function declaration
    • Code block: where the instructions for defining the function (and what it should do) lives. It is also referred to as the function body
    • Console Method: Steps in the code block are written through a console.log
    • The word console refers to the test environment that developers use to check out their code
    • The second part .log() is a method, which means a set of instructions that can be execurtes by a computer
    • Console.logs output a message to the web console by adding an argument located inside the parenthesis
    • Define after the computer logs to the console

  2. Call the function
    • To call the function use the function declaration followed by a set of parenthesis and a semicolon
    • When function is called properly, the instructions listed in the code block are executed

Variables: a named container that allows to stores data in our code


Coding a Variable


  1. Declaring a variable, the "var" keyword is used. This works for conditional statements and arrays.
  2. After the "var", give the variable a unique name. This is how it weill be referenced in our code
  3. Finally, an assignment opperator or = is used to assign the specific data that is being stored
  4. === is a strict equality operator which checks to see if two values are equal and returns to the Boolean result of true or false if the values are not equal

Arrays: stores a group of related data in a variable and uses a for loop to preform a task on each piece of data, while only using a few lines of code.


Coding an Array


  1. In order to use an array with a single piece of data, the location of the data item within the array is neccessary. This location is identified with a unique number called an index.
  2. To access an individual index, start with the name of the array, then a pair of square brackets and inside is the number of the index, to see the array in action it is put in the console.log function
  3. To preform a for loop a conditional statement like an if or else statement has to be used as well
  4. The second half needed for a for loop is the array's length. This determines the amount of times the for loop is repeated until there are no items left in the array.

Three statements in a for loop


  1. Starting point of a for loop: declare the variable (x) and give it a value of 0 (all first indices begin with 0)
  2. Conditon of a for loop: as long as the number in the variable x is less than the length of the array, the loop will continue running. This will allow the loop to execute one time for each array. To do this, we use the .length property within our array.
  3. The final statement: allows the array to reiterate over each item. To show the increse of the indices by 1, use the shortcut x++. This allows the code to loop while increasing by 1, without having to type all the console.logs

Terms


  • Control Flow: is the order in which a computer executes in a script, which runs from the first statement at the top down to the bottom of the page. Can be useful by using conditional statements and loops
  • Conditional Statements: allows programs to follow written instructions based on the variables
  • If statements: checks whether or not a condition is true or false
  • Truthy: If condition is true. If a conditional statement is truthy it will follow the action we asked it to do in between curly brackets. If it is not truthy, or considered falsy, then the computer will ignore the conditional statement between the brackets.
  • Falsy: it will be denoted as 0, -0, On, "", null, undefined, NaN, or a Boolean data type of false.
  • Else if: a different conditional statement that can be used if the second contion is truthy
  • Loops: allow for indices to be stated without having to list them all within a console log
  • Index: Each piece of a data's location number can access the individual data inside the array. JavaScript and many other programming languages have a 0 index and are sequencial following the 0. Meaning the data index will begin with the number 0, then 1, 2, 3...
  • for loops: uses predictable pattern of indices to preform a task on all the items in an array by allowing a single code block to be executed over and over.
  • Browser Application Programming Interfaces (API's): provide functionality such as: dynamically creating HTML and setting CSS styles; collecting and manipulating a video stream from a user's webcam, or generating 3D graphics and audio samples
  • Third-party API's: allow developers to incorporate functionality in sites from other content providers, such as Twitter or Facebook
  • Third-party frameworks and libraries: can apply to HTML to accelerate the work of building sites and applications
